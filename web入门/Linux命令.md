# 简单运算符

## 1️⃣ `&`

**作用：把命令放到后台执行**

示例：

`sleep 10 &`

✅ 这会让 `sleep 10` 在后台运行，你可以继续在当前终端输入其他命令。

> 适合长时间运行的任务，比如下载、备份等。

---

## 2️⃣ `&&`

**作用：逻辑与，只有前一个命令成功时才执行后一个命令**

示例：

`mkdir test && cd test`

✅ 如果 `mkdir test` 成功，就进入 `test` 目录；如果 `mkdir` 失败，`cd test` 就不会执行。

> 常用于写安全的脚本，避免后续命令在前一步失败时继续执行。

---

## 3️⃣ `>`

**作用：输出重定向，把命令的结果写入文件（覆盖原文件）**

示例：

`echo "hello" > file.txt`

✅ 如果 `file.txt` 存在，会被覆盖；如果不存在，会创建新文件。

---

## 4️⃣ `>>`

**作用：追加重定向，把命令的结果追加到文件末尾**

示例：

`echo "world" >> file.txt`

✅ 不会覆盖文件，而是把 `"world"` 加到文件末尾。

# ssh 
```bash
ssh 用户名@远程主机IP

```

这样就可以直接远程操控别人ip的主机

## 🔧 常用参数

|参数|作用|
|---|---|
|`-p`|指定远程端口（默认 22）|
|`-i`|指定私钥文件，用于免密码登录|
|`-v`|输出详细调试信息，便于排错|
## ⚡ 举例

1. **普通登录**

`ssh alice@203.0.113.5`

2. **使用非默认端口**

`ssh -p 2222 alice@203.0.113.5`

3. **用私钥登录**

`ssh -i ~/.ssh/id_rsa alice@203.0.113.5`

**删除文件和文件夹 （rm）**


`rm` 在我们迄今为止介绍的命令中非常出色。您可以使用 `rm` 简单地删除文件。但是，删目录要 `-R` 。


**复制和移动文件和文件夹 （cp、mv）**


`cp` 将现有文件的全部内容复制到新文件中。在下面的屏幕截图中，我们将“note”复制到“note2”。
```bash
tryhackme@linux2:~$ cp note note2 
tryhackme@linux2:~$ ls folder1 note note2
```


移动文件需要两个参数，就像 cp 命令一样。但是，`mv` 不会复制和/或创建新文件，而是合并或修改我们作为参数提供的第二个文件。您不仅可以使用 `mv` 将文件移动到新文件夹，还可以使用 `mv` 重命名文件或文件夹。例如，在下面的屏幕截图中，我们将文件“note2”重命名为“note3”。“note3”现在将具有“note2”的内容。


```bash
tryhackme@linux2:~$ mv note2 note3 


tryhackme@linux2:~$ ls folder1 note note3

```




# 权限

`su`可以切换用户，
`su point`输入point的密码就可以登录

## 常用参数

| 参数         | 作用                                     |
| ---------- | -------------------------------------- |
| `-` 或 `-l` | 切换用户并加载该用户的环境变量（如 PATH、HOME）           |
| `-c "命令"`  | 以指定用户身份执行一条命令，例如：`su -c "whoami" root` |

### 实用示例

1. 切换到 root 并加载 root 环境：

2. 以 root 身份执行单条命令：
    

`su -c "apt update" root`

3. 切换到另一个用户 alice：
    

`su - alice`

## nano

`nano` 是 Linux / macOS 系统中一个 **轻量级文本编辑器**，常用于在终端里编辑文件。

可以把它理解为 **命令行下的记事本**

### 基本用法

`nano test.txt`

### 🔹 Vim 的模式

`vim` 的核心是 **模式切换**，主要有三种：

|模式|功能|
|---|---|
|**普通模式（Normal）**|默认模式，用于浏览文件、删除、复制、粘贴、跳转|
|**插入模式（Insert）**|用于输入文本（按 `i` 进入插入模式）|
|**命令模式（Command）**|用于保存、退出、查找替换等（按 `:` 进入命令模式）|

---

### 🔹 基本操作

#### 1. 进入插入模式

`i  → 在光标前插入 I  → 行首插入 a  → 在光标后插入 A  → 行尾插入`

#### 2. 返回普通模式

`Esc`

#### 3. 保存与退出

在普通模式下：

`:w   → 保存 :q   → 退出 :wq  → 保存并退出 :q!  → 强制退出不保存`

#### 4. 文本操作（普通模式）

`dd  → 删除整行 yy  → 复制整行 p   → 粘贴 u   → 撤销 Ctrl+r → 重做`

#### 5. 查找与替换

`:/关键字   → 查找关键字 :%s/旧/新/g → 替换整个文件的内容`


## wget

简单来说，`wget` 可以在终端里下载网页、文件或者整个网站，而不需要打开浏览器。

---

### 🔹 基本用法

`wget URL`

### 示例：

`wget https://example.com/file.zip`

- 会把 `file.zip` 下载到当前目录
    

---

### 🔹 常用参数

|参数|作用|
|---|---|
|`-O 文件名`|指定下载后的文件名|
|`-P 目录`|指定保存的目录|
|`-c`|断点续传，如果下载中断可以继续|
|`-q`|静默模式，不输出信息|
|`-r`|递归下载整个网站|
|`--no-check-certificate`|忽略 SSL 证书验证（不推荐长期使用）|

---

### 🔹 示例操作

1. 下载并指定文件名：
    

`wget -O myfile.zip https://example.com/file.zip`

2. 下载到指定目录：
    

`wget -P ~/Downloads https://example.com/file.zip`

3. 断点续传：
    (下一半，继续下)

`wget -c https://example.com/largefile.iso`

4. 递归下载网站：
    **意思**：`wget` 不仅下载你指定的页面或文件，还可以 **自动下载该网页中链接到的内容**，甚至是整个网站。

`wget -r https://example.com/`
- 会下载网页上的 HTML、图片、CSS、JS 等资源
    
- 目录结构会保持原网站结构



# scp 

安全复制
与常规 cp 命令不同，此命令允许您使用 SSH 协议在两台计算机之间传输文件，以提供身份验证和加密。



在 SOURCE 和 DESTINATION 模型上工作，SCP 允许您：
- 将文件和目录从当前系统复制到远程系统
- 将文件和目录从远程系统复制到当前系统


## 1️⃣ 基本概念

在 `scp` 命令里，用户的格式通常是：

`用户名@远程主机:路径`

- **用户名**：你想访问远程主机的账号
    
- **远程主机**：IP 地址或域名
    
- **路径**：文件或目录在远程主机上的位置
    

> 用户名告诉远程主机你以哪个身份去登录，这决定了你的权限（能否读写文件）。

---

## 2️⃣ 基本用法示例

### 示例 1：本地文件复制到远程

`scp file.txt alice@192.168.1.100:/home/alice/`

- `file.txt`：本地要上传的文件
    
- `alice`：远程用户
    
- `/home/alice/`：远程存放路径
    

> 系统会要求输入 **alice 用户的密码**（除非配置了 SSH 密钥免密码登录）。

---

### 示例 2：从远程复制文件到本地

`scp bob@192.168.1.100:/home/bob/data.txt ./`

- `bob`：远程主机用户名
    
- `/home/bob/data.txt`：远程文件路径
    
- `./`：当前本地目录
    

---

### 示例 3：复制目录（递归）到远程

`scp -r myfolder charlie@192.168.1.100:/home/charlie/`

- `-r`：递归复制目录
    
- `charlie`：远程用户
    
- `/home/charlie/`：远程目录
    

---

## 3️⃣ 关于用户的注意事项

1. **权限决定能否写入**
    
    - 如果远程用户没有目标目录的写权限，上传会失败
        
    - 例：
        
        `scp file.txt user@192.168.1.100:/root/`
        
        如果 `user` 不是 root，会提示权限不足
        
2. **SSH 密钥登录**
    
    - 可以通过 SSH 密钥免密码登录远程用户：
        
        `scp -i ~/.ssh/id_rsa file.txt alice@192.168.1.100:/home/alice/`
        
3. **不同用户权限**
    
    - 不同用户对应不同的家目录和文件权限
        
    - 在渗透测试中，如果远程用户是 root，可以操作系统中几乎所有文件


# curl
直接终端访问web

## 🔹 常用参数

|参数|作用|
|---|---|
|`-O`|保存下载文件，保持原文件名|
|`-o 文件名`|保存下载文件，并指定文件名|
|`-L`|跟随重定向（HTTP 3xx）|
|`-I`|只显示响应头（HEAD 请求）|
|`-X`|指定请求方法（GET、POST、PUT、DELETE）|
|`-d`|POST 数据发送|
|`-H`|添加请求头（Headers）|
|`-u 用户:密码`|HTTP 认证登录|

---

## 🔹 常用示例

1. 下载文件并保持原名：
    

`curl -O https://example.com/file.zip`

2. 下载文件并指定保存名称：
    

`curl -o myfile.zip https://example.com/file.zip`

3. 发 POST 请求：
    

`curl -X POST -d "username=alice&password=123" https://example.com/login`

4. 添加自定义请求头：
    

`curl -H "Authorization: Bearer TOKEN" https://example.com/api/data`

5. 查看响应头：
    

`curl -I https://example.com`
